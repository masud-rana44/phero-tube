Questions

1. Discuss the scope of var, let, and const

In JavaScript, var, let, and const are used for declaring variables. Their scopes are different:

  • var: Function-scoped. Variables declared with var are hoisted to the top of their function or global context.
  • let: Block-scoped. Variables declared with let are only accessible within the block they are declared in.
  • const: Block-scoped. Similar to let, but const variables cannot be reassigned after declaration.

2. Tell us the use cases of null and undefined

null: It represents an intentional absence of any object value. Use it when you want to indicate that a variable should have no value.

undefined: It typically means a variable has been declared but hasn't been assigned a value yet. It's the default value of function parameters.

3. What do you mean by REST API?

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building and interacting with web services. It uses HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources, which are represented as URLs. RESTful APIs are stateless and rely on standard HTTP methods like GET, POST, PUT, and DELETE for communication.
